**Milestone 1: Use Case DSL**
- [ ] `UseCase`, `BaseUseCase`, `UseCaseContext`, `VoidVerification`, `ResponseVerification`, and `ErrorVerification` are defined
- [ ] `UseCaseDsl` is the entry point with one lazy accessor per domain
- [ ] `ShopDsl` has one factory method per use case; data-returning use cases have a paired `Verification` class
- [ ] `BaseUseCaseDslTest` exposes `app` and no longer exposes raw drivers

**Milestone 2: System Smoke Tests**
- [ ] All system smoke tests use `app.shop().<useCase>().execute().shouldSucceed()` style

**Milestone 3: External System Smoke Tests**
- [ ] Each external system has a DSL class following the same pattern as `ShopDsl`
- [ ] All external system smoke tests call through the DSL

**Milestone 4: E2E Tests**
- [ ] Use case classes exist for each shop operation
- [ ] Data-returning use cases have paired `Verification` classes
- [ ] Aliases thread system-generated values between use cases
- [ ] All E2E tests use DSL use case calls — no manual request construction or direct driver references
