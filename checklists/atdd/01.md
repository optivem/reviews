**Milestone 1: Setup**

- [ ] Project repository includes yourself and any team members

**Milestone 2: API CRUD**

- [ ] REST API has at least one entity with create and retrieve endpoints
- [ ] Entity is persisted to a database
- [ ] API responds correctly when called manually

**Milestone 3: UI CRUD**

- [ ] UI has a form to create a new entity
- [ ] UI has a page to view existing entities
- [ ] UI calls the API — no direct database access
- [ ] Creating an entity through the UI and viewing it works end-to-end

**Milestone 4: External API Dependency**

- [ ] System depends on at least one external REST API
- [ ] External API is callable and returns data
- [ ] System calls the external API during a normal operation (not just startup)
- [ ] External API starts as part of docker-compose

**Milestone 5: Clock Dependency**

- [ ] System depends on a clock abstraction (not a direct system clock call)
- [ ] At least one operation uses the clock (e.g. timestamps, time-based logic)
- [ ] Clock is callable and returns the current time

**Milestone 6: Business Logic**

- [ ] At least one calculation that uses data from an external system
- [ ] At least one validation rule that rejects invalid input with a clear error
- [ ] At least one status or state transition
- [ ] All logic is verifiable through the API (and visible in the UI where applicable)

**Milestone 7: Manual Test**

- [ ] System starts and UI loads in a browser
- [ ] API responds correctly
- [ ] Primary use case works end-to-end (positive flow via API and UI)
- [ ] At least one validation rule rejects invalid input with a clear error (negative flow)

**Milestone 8: System Architecture Diagram**

- [ ] System Architecture Diagram created (shows components, external systems, and relationships)

**Milestone 9: System Behavior Documentation**

- [ ] Primary actors identified
- [ ] Use cases identified
- [ ] Secondary actors (external systems) identified, including at least one I/O external system
- [ ] Use Case Diagram created
- [ ] Use Case Narrative written for at least one use case
- [ ] Documentation published (GitHub Pages or README)
