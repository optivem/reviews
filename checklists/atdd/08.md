**Task 1: Smoke Tests**

- [ ] `ScenarioDsl` interface is defined with `assume()`, `given()`, and `when()`
- [ ] `AssumeStage` has one method per system, each returning `AssumeRunning`
- [ ] `ScenarioDslImpl` wraps `UseCaseDsl` and implements `ScenarioDsl`
- [ ] `BaseScenarioDslTest` exposes `scenario` — no raw `app` or driver fields
- [ ] All smoke tests use `scenario.assume().*().shouldBeRunning()`


**Task 2: E2E Tests**

- [ ] `GivenStage` has one method per business precondition concept
- [ ] Each Given step has `with*()` configuration methods and extends `GivenStep`
- [ ] `WhenStage` has one method per action
- [ ] Each When step has `with*()` configuration methods and `execute()` via `WhenStep`
- [ ] `ThenResultStage` exposes `shouldSucceed()` and `shouldFail()`
- [ ] Domain Then steps (e.g. `ThenOrder`) use `has*()` naming for assertions
- [ ] `given()` and `when()` guard against multiple executions per test method
- [ ] Each stage implementation delegates to the corresponding Use Case DSL use case
- [ ] All E2E tests use `scenario.given()...when()...then()` — no Use Case DSL calls
