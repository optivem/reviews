**Milestone 1: Scenario DSL Port**
- [ ] `ScenarioDsl` interface is defined with `assume()`, `given()`, `when()`
- [ ] `AssumeStage` has one method per system, each returning `AssumeRunning`
- [ ] `GivenStage`, `WhenStage`, `ThenResultStage` and domain step interfaces are defined

**Milestone 2: Scenario DSL Core**
- [ ] `ScenarioDslImpl` wraps `UseCaseDsl` and implements the assume stage
- [ ] Given, When, and Then steps are implemented in `dsl-core`
- [ ] `given()` and `when()` guard against multiple executions per test method
- [ ] `BaseScenarioDslTest` exposes `scenario` — no `app` or raw driver fields

**Milestone 3: System Smoke Tests**
- [ ] All system smoke tests use `scenario.assume().<system>().shouldBeRunning()`

**Milestone 4: External System Smoke Tests**
- [ ] All external system smoke tests use `scenario.assume().<system>().shouldBeRunning()`

**Milestone 5: E2E Tests**
- [ ] All E2E tests use `scenario.given()...when()...then()` — no Use Case DSL calls
